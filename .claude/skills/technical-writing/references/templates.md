# Document Templates

## Design Doc (問題解決型)

プロジェクトの問題定義から解決策までを記述する。

```markdown
# 1. 概要
[1-2 段落で全体像。問題、解決策、期待効果を簡潔に]

# 2. 背景

## 2.1 コンテキスト
[システムの位置づけ、関連システム、ステークホルダー]

## 2.2 現状の仕組み
[既存プロセス/システムの説明]

# 3. 問題

## 3.1 課題
[課題と根拠データを簡潔に。詳細分析は別レポート参照可]

## 3.2 目標と KPI
| 指標 | 現状 | 目標 | 測定方法 |
|------|------|------|----------|
| ... | ... | ... | ... |

## 3.3 スコープ外
[今回対象外とする事項とその理由]

# 4. 解決策

## 4.1 アプローチ
[採用する解決策とその理由]

## 4.2 実装計画
[具体的な実装ステップ]

## 4.3 代替案
[検討したが採用しなかった案とその理由]

# 5. セキュリティ
[認証・認可、データアクセス制御、脆弱性対策]

# 6. プライバシー
[個人情報の取り扱い、データ保持期間、同意取得]

# 7. 可観測性
[ログ設計、メトリクス、アラート条件]

# 8. パフォーマンス
[レイテンシ要件、スループット、負荷見積もり]

# 9. 障害対応
[エラー時の挙動、リトライ戦略、ロールバック手順]
```

**注**: セクション 5-9 は必要な場合のみ記載。
